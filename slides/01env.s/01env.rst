コードリーディングにかかわる環境
-------------------------------------------------------------------------

ビルド環境
  ソースコードから実行ファイルを構築するのに使う環境

実行環境
  実行ファイルを実行する環境

調査環境
  演習参加者がソースコードを調査する環境

----

=> 本演習ではGNU/Linuxディストリビューションの一つFedoraを使う。

* 演習担当者が常用している。
* ソースコードの入手が整備されている。
 
FedoraとGNU/Linuxディストリビューション
-------------------------------------------------------------------------
* Linuxカーネルを中心に様々なフリーソフトウェア/オープンソースソ
  フトウェア(FOSS)によって構成されている。

.. figure:: ../../fedora.svg
	:scale: 150%

* アップストリームプロジェクトが提供するソースコードがビルドされて
  簡単にインストールできる形式(パッケージ)で提供されている。

* パッケージをインストール、アンインストール、アップデートするための
  コマンド(パッケージシステム)が用意されている。

* Fedoraの場合 yum及びrpm コマンドでパッケージを操作できる。

他の環境
-------------------------------------------------------------------------
* パッケージシステムを経由してソースコードを入手するところを除いて、
  他のGNU/Linuxディストリビューションを調査環境に用いても、演習で得た知見
  はそのまま有効はなずである。

* Window, Mac OSXなどでは、ツールの使い型といった知見は役に立たないかもしれ
  ない。ただしコードリーディングの基本的な考え方はある程度有効なはずである。

* 以降ではFedoraを前提に説明をする。


パッケージ管理
------------------------------------------------------------------------

* yumあるいはrpmコマンドを使うと、様々なソフトウェアを即実行可能な形で
  でインストールできる。
* ソフトウェアをパッケージ(拡張子rpm)形式で入手、インストールできる。
  
  - Fedoraプロジェクトの管理するウェブサーバから入手する。
  - 次に述べるソースパッケージと区別するためにバイナリパッケージと呼ぶ場合がある。
    
* ソフトウェアのソースコードもパッケージとして提供されている。

  - ソースコードを梱包したパッケージを特にソースパッケージ(拡張子src.rpm) 
    と呼ぶ。

* 本演習で使うツールについてバイナリパッケージとしてインストールできる。
* このスライドでは、バイナリパッケージの操作を説明する。

パッケージ管理: 検索
------------------------------------------------------------------------
パッケージをインストールするにはその名前を知っている必要がある。

キーワードを用いた入手可能なパッケージの検索::

	$ yum search WORD

パッケージの詳細な情報の確認::

	$ yum info パッケージ名

----

* デモ

::

	$ yum search emacs
	$ yum info emacs

パッケージ管理: インストール
------------------------------------------------------------------------
* 名前を指定してパッケージをインストールする。
* インストールには管理者権限が必要である。
* あるパッケージを利用するのに必要な他のパッケージもあわせて自動的に
  インストールされる。

::

	$ su 
	Password: 
	# yum install パッケージ名
	# exit

----

* デモ

::

	$ yum install emacs

パッケージ管理: 問合せ
------------------------------------------------------------------------
rpmを使うとシステムにインストールされたパッケージについて問合せを
かけることができる。

インストールされているパッケージの一覧
	rpm -qa
あるパッケージPに含まれるファイルの一覧
	rpm -ql P
あるファイルFが所属するパッケージの名前
	rpm -qf F
あるパッケージPの説明
	rpm -qi P
